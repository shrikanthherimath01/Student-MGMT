HOW TO CHANGE WHATSAPP SENDER NUMBER
Complete Guide - Delete Old & Connect New WhatsApp
📌 WHAT THIS DOES
Action	Description
Deletes	Old WhatsApp session files
Removes	Previous number connection
Allows	New QR code scan with different number
Requires	No code changes
🔧 ALL COMMANDS WITH COMMENTS
STEP 1: Stop the Bridge
powershell
# Go to terminal where node server.js is running
# Press these keys together:
Ctrl + C

# This stops the WhatsApp bridge service
STEP 2: Navigate to Project Folder
powershell
# Change directory to your SMS folder
cd C:\StudentManagementSystem\sms

# Comment: This is where your WhatsApp bridge files are located
STEP 3: Delete Old WhatsApp Session
powershell
# Delete the main session folder (where WhatsApp login is stored)
Remove-Item -Path "whatsapp-session" -Recurse -Force -ErrorAction SilentlyContinue

# Comment: This removes the saved WhatsApp login from old number
powershell
# Delete backup/auth folder (if exists)
Remove-Item -Path ".wwebjs_auth" -Recurse -Force -ErrorAction SilentlyContinue

# Comment: This removes any remaining authentication files
STEP 4: Verify Deletion (Optional)
powershell
# Check if folders are gone
dir whatsapp-session -ErrorAction SilentlyContinue

# Comment: If nothing shows, deletion was successful
STEP 5: Restart WhatsApp Bridge
powershell
# Start the bridge with clean session
node server.js

# Comment: This will generate a NEW QR code for scanning
STEP 6: Connect New WhatsApp Number
powershell
# After QR code appears in terminal:

# 1. Open WhatsApp on your NEW phone number
# 2. Tap Settings (3 dots or gear icon)
# 3. Tap Linked Devices
# 4. Tap Link a Device
# 5. Scan the QR code from terminal

# Comment: The number that scans becomes the new sender
STEP 7: Verify Connection
powershell
# Open browser and go to:
http://localhost:3001/status

# Expected response:
# {"connected":true,"qr":null}

# Comment: "connected":true means new number is active

🔍 TROUBLESHOOTING COMMANDS
Check if bridge is running
powershell
# In browser or run:
curl http://localhost:3001/status

# Comment: If connection refused, bridge is not running
Force kill stuck processes
powershell
# Kill all Node processes
taskkill /F /IM node.exe

# Kill all Chrome processes  
taskkill /F /IM chrome.exe

# Comment: Use if bridge won't stop or restart properly
Check what's on port 3001
powershell
# See which process is using port 3001
netstat -ano | findstr :3001

# Comment: Shows PID of process on port 3001
Delete session if permission denied
powershell
# Force delete with different method
rm -r -force whatsapp-session -ErrorAction SilentlyContinue
rm -r -force .wwebjs_auth -ErrorAction SilentlyContinue

# Comment: Alternative deletion command
